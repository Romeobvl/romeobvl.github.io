C# SQL Server n8n Workflows ERP EBP FTP / VPN

Développement de services Windows en C# et de workflows d'automatisation pour l'intégration de commandes e-commerce vers des ERP clients (EBP). Missions en autonomie sur des projets clients réels.

19 – 23 jan. Projet Grasset

Service Windows C# — Import CSV vers ERP

Découverte de l'environnement : C#, Visual Studio Code, SQL Server et n8n. Mission principale : créer un service Windows qui récupère des commandes du site AM Shop via une requête SQL, les transforme en fichier CSV et les dépose sur un serveur FTP pour alimenter l'ERP du client Grasset.

Un service Windows expose deux méthodes par défaut (OnStart / OnStop). J'ai ajouté un timer en ms déclenchant OnTimer pour exécuter la logique à intervalles réguliers. Finalisation de la requête SQL, création de la fonction createFile, connexion BDD SQL, compilation de l'exécutable et installation via InstallUtil.exe.

22 jan. – 6 fév. Projet SPASSO

Import factures multi-marketplaces vers ERP EBP

SPASSO agrège les commandes de plusieurs marketplaces (Bricodépôt, Castorama, Cdiscount, Leroy Merlin, ManoMano…). Aptitudes Méditerranée gère un service d'import vers l'ERP EBP. Problème : les fichiers fournis sont souvent mal formatés, nécessitant un fichier de configuration de cas particuliers.

Missions réalisées : conversion des montants TTC en HT (et inversement) selon le pays et la marketplace, gestion des TVA par dépôt (ex. Italie 22%), remises spécifiques par client, livraison d'une mise à jour n8n et des requêtes SQL le 02/02 avec appel client en parallèle.

11 – 22 fév. Workflow n8n — JSON → ERP EBP

Création d'un workflow n8n complet from scratch

Décision de centraliser tous les workflows clients chez Aptitudes Méditerranée (et non sur les VMs clients). Création d'un workflow complet : réception d'un JSON via Webhook, procédure SQL pour récupérer le nom du client (absent du JSON), transformation et uniformisation des données, conversion en CSV, puis import dans l'ERP EBP via l'API EBP.

Le workflow gère l'éclatement des articles (Split Out), la sélection des colonnes utiles, le paramétrage de l'API EBP (URL, clé, masque d'import, table cible) et la gestion d'erreur avec réponse Webhook. Un bloc Set CALL API EBP1 final gère l'import des commandes.
